```
┌──────────────────────────────────────────────────┐
│                  Mobile App                       │
│  apps/mobile/  (Vite + React + TanStack Router)  │
│                                                   │
│  ┌─────────┐  ┌─────────┐  ┌─────────────────┐  │
│  │ Connect  │  │ Devices │  │   Chat View     │  │
│  │ (QR/URL) │→ │ & Sess. │→ │  (messages,     │  │
│  │          │  │  List   │  │   composer,      │  │
│  │          │  │         │  │   approvals,TTS) │  │
│  └─────────┘  └─────────┘  └─────────────────┘  │
│         │            │              │             │
│         └────────────┴──────────────┘             │
│                      │                            │
│              ┌───────┴────────┐                   │
│              │  WsTransport   │                   │
│              │  (per device)  │                   │
│              └───────┬────────┘                   │
└──────────────────────┼───────────────────────────┘
                       │ WebSocket over Tailscale
        ┌──────────────┼──────────────┐
        ▼              ▼              ▼
   ┌─────────┐   ┌─────────┐   ┌─────────┐
   │ Desktop  │   │ Laptop  │   │ Server  │
   │  :3773   │   │  :3773  │   │  :3773  │
   └─────────┘   └─────────┘   └─────────┘
```

### Key Principles

- **Phone is client-only** — never runs a Kuumba Code server, never appears in anyone's device list
- **All devices visible simultaneously** — no "Local vs Remote" tab needed; everything is remote from the phone's perspective
- **No file explorer, no diff viewer, no terminal, no code editor** — just the conversation, approvals, and TTS
- **Authentication via QR code scan** — phone scans QR from Settings on any desktop, gets connection string with auth token
- **Copy components, don't share packages** — avoids monorepo coupling and merge conflicts with upstream

**Purpose:** First-time device setup. Scan QR code from desktop Settings page or enter connection details manually.

**Layout:**
```

│  │                       │  │
│  └───────────────────────┘  │
│                             │
│  ─── or enter manually ───  │  ← Divider
│                             │
│  Device name:  [________]   │
│  Host:         [________]   │  ← Tailscale hostname
│  Port:         [__3773__]   │
│  Auth token:   [________]   │
│                             │
│  [ Connect & Save Device ]  │  ← Primary button
│                             │
└─────────────────────────────┘
```

**Components used:**
- `Input` (4 fields)
- `Button` (connect)
- Capacitor `BarcodeScanner` plugin (native) or fallback text input (browser)

**Behavior:**
1. QR scan decodes JSON: `{ host, port, token, deviceName? }`
2. App creates a test WebSocket connection to validate
3. On success: saves device to `settingsStore`, navigates to `/devices`
4. On failure: shows error toast, stays on page

**Browser fallback (pre-Capacitor):**
- QR scanner area shows a paste field: "Paste connection string from desktop"
- User copies JSON from DeviceQRCode component on desktop, pastes here

## 6. State Management

### Zustand Stores

#### `devicesStore.ts` — Device discovery and session state

```ts
interface DeviceState {
  deviceId: string;          // From saved config
  config: SavedDevice;       // { name, host, port, authToken }
  online: boolean;
  lastSeen: number | null;
  deviceName: string;        // From /api/device-info response
  sessions: RemoteSession[]; // Active shared sessions
}

interface RemoteSession {
  threadId: string;
  projectId: string;
  projectName: string;
  projectCwd: string;
  status: "idle" | "running" | "ready" | "error";
  title: string;
}

interface DevicesStore {
  devices: Record<string, DeviceState>;  // keyed by deviceId
  refreshing: boolean;

  // Actions
  refreshAll: () => Promise<void>;
  refreshDevice: (deviceId: string) => Promise<void>;
  addDevice: (config: SavedDevice) => void;
  removeDevice: (deviceId: string) => void;
}
```

#### `connectionStore.ts` — Active WebSocket connections

```ts
interface ActiveConnection {
  deviceId: string;
  transport: WsTransport;
  api: NativeApi;
  status: "connecting" | "connected" | "error" | "closed";
  error?: string;
}

interface ConnectionStore {
  connections: Record<string, ActiveConnection>;  // keyed by deviceId

  // Actions
  connect: (deviceId: string, config: SavedDevice) => Promise<NativeApi>;
  disconnect: (deviceId: string) => void;
  disconnectAll: () => void;
  getApi: (deviceId: string) => NativeApi | null;
}
```

#### `settingsStore.ts` — Persisted app preferences

```ts
interface MobileSettings {
  theme: "system" | "light" | "dark";
  ttsSpeed: 1 | 1.5 | 2;
  ttsAutoDownload: boolean;
  savedDevices: SavedDevice[];
}

interface SavedDevice {
  id: string;         // UUID generated on save
  name: string;       // User-friendly name
  host: string;       // Tailscale hostname
  port: number;       // Server port (default 3773)
  authToken: string;  // T3CODE_AUTH_TOKEN value
}
```

### Persistence

- `settingsStore` persisted to `localStorage` (browser) / Capacitor `Preferences` plugin (native)
- `devicesStore` and `connectionStore` are ephemeral (refreshed on app open)

## 7. WebSocket & API Layer

### Connection Strategy

The mobile app manages **multiple simultaneous connections** — one per device. Unlike the desktop app which swaps a single API globally, the mobile app keeps a pool of connections.

```ts
// lib/nativeApi.ts — Simplified for mobile
import { WsTransport } from "./wsTransport";
import { WS_METHODS, WS_CHANNELS, type NativeApi } from "@t3tools/contracts";

export function createMobileNativeApi(wsUrl: string): NativeApi {
  const transport = new WsTransport(wsUrl);

  return {
    // Only implement the methods mobile actually uses:
    dialogs: {
      pickFolder: async () => null,     // N/A on mobile
      confirm: async (msg) => window.confirm(msg),
    },
    terminal: { /* no-op stubs — mobile doesn't use terminal */ },
    projects: {
      searchEntries: (input) => transport.request(WS_METHODS.projectsSearchEntries, input),
      writeFile: (input) => transport.request(WS_METHODS.projectsWriteFile, input),
      readFile: (input) => transport.request(WS_METHODS.projectsReadFile, input),
    },
    shell: {
      openInEditor: async () => {},     // N/A on mobile
      openExternal: async (url) => { window.open(url, "_blank"); },
    },
    git: { /* passthrough to transport — read-only on mobile */ },
    contextMenu: {
      show: async () => null,           // N/A on mobile
    },
    sessions: {
      setRemoteSharing: (input) => transport.request(WS_METHODS.sessionsSetRemoteSharing, input),
      getRemoteSharing: (input) => transport.request(WS_METHODS.sessionsGetRemoteSharing, input),
    },
    server: {
      getConfig: () => transport.request(WS_METHODS.serverGetConfig),
      upsertKeybinding: async () => ({ configPath: "", validKeybindings: [] }),
    },
    orchestration: {
      getSnapshot: () => transport.request(ORCHESTRATION_WS_METHODS.getSnapshot),
      dispatchCommand: (command) =>
        transport.request(ORCHESTRATION_WS_METHODS.dispatchCommand, { command }),
      getTurnDiff: (input) => transport.request(ORCHESTRATION_WS_METHODS.getTurnDiff, input),
      getFullThreadDiff: (input) =>
        transport.request(ORCHESTRATION_WS_METHODS.getFullThreadDiff, input),
      replayEvents: (fromSeq) =>
        transport.request(ORCHESTRATION_WS_METHODS.replayEvents, { fromSequenceExclusive: fromSeq }),
      onDomainEvent: (callback) =>
        transport.subscribe(ORCHESTRATION_WS_CHANNELS.domainEvent, (msg) => callback(msg.data)),
    },
  };
}
```

### WebSocket URL Construction

```ts
function buildWsUrl(device: SavedDevice): string {
  const proto = "ws"; // Tailscale is already encrypted
  const tokenParam = device.authToken ? `?token=${encodeURIComponent(device.authToken)}` : "";
  return `${proto}://${device.host}:${device.port}${tokenParam}`;
}
```

### Device Discovery (HTTP polling)

```ts
// Reuse the same pattern as apps/web/src/remoteDevices.ts
async function fetchDeviceInfo(device: SavedDevice): Promise<DeviceInfo | null> {
  try {
    const res = await fetch(`http://${device.host}:${device.port}/api/device-info`, {
      signal: AbortSignal.timeout(5000),
      headers: device.authToken ? { Authorization: `Bearer ${device.authToken}` } : {},
    });
    if (!res.ok) return null;
    return await res.json();
  } catch {
    return null;
  }
}

## Risk Mitigation

| Risk | Mitigation |
|------|-----------|
| Piper WASM doesn't load in Capacitor WebView | Test early in Phase 2C-5. Fallback: browser SpeechSynthesis API |
| WebSocket connection drops on mobile sleep | Reconnection logic in WsTransport (exponential backoff already built) |
| Tailscale not reachable from phone | Show clear error with troubleshooting steps. Verify Tailscale app is active. |
| Large message threads cause scroll jank | Limit initial load to last 50 messages, lazy-load older on scroll up |
| Mixed content (HTTP over Tailscale) blocked | Capacitor `allowMixedContent: true` in config. Test early. |
| Voice model download fails on mobile data | Show file size warning. Allow Wi-Fi-only download setting. |
| Code highlighting is slow on mobile | Use lighter Shiki theme. Limit highlighting to first 200 lines per block. |
